1. Default Parameter : Default Parameter in JavaScript allows us to assign a default value to a function parameter if no value or undefined is passed during the function call. It helps avoid undefined values and makes the function safer and cleaner.
e.g. : 
function greet(name = "Guest") {
    console.log("Hello " + name);
}

greet(); // Hello Guest
greet("Prabhav"); // Hello Prabhav


-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

2. Spread Operator (...) : The Spread Operator is used to expand elements of an array, object, or iterable into individual elements. It is useful for copying arrays, merging arrays, and cloning objects.

e.g :
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5];

console.log(arr2); // [1, 2, 3, 4, 5]

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


3. Rest Operator (...) : The Rest Operator collects multiple elements and combines them into a single array or object. It is mainly used in function parameters and destructuring.

e.g. : 

const [first, ...rest] = [10, 20, 30, 40];

console.log(first); // 10
console.log(rest);  // [20, 30, 40]

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


4. Rest Parameter : Rest Parameter allows a function to accept an unlimited number of arguments and stores them in an array. It improves flexibility in function design.

e.g. : 
function sum(...numbers) {
    return numbers.reduce((acc, curr) => acc + curr, 0);
}

console.log(sum(10, 20, 30)); // 60


-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

5. Destructuring : Destructuring is a JavaScript feature used to extract values from arrays or objects and assign them to variables in a shorter and cleaner way. 

e.g. : 
const person = {
    name: "Prabhav",
    age: 22
};

const { name, age } = person;

console.log(name); // Prabhav
console.log(age);  // 22

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

6. Position Based Extraction : Position Based Extraction is used in Array Destructuring where values are extracted based on their position (index) in the array.

e.g. : 
const colors = ["Red", "Blue", "Green"];

const [first, second] = colors;

console.log(first);  // Red
console.log(second); // Blue

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


7. Key Based Extraction : Key Based Extraction is used in Object Destructuring where values are extracted using object property names (keys), not positions.

e.g. : 
const student = {
    name: "Prabhav",
    course: "CSE"
};

const { name, course } = student;

console.log(name);   // Prabhav
console.log(course); // CSE

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

8. Async API : Async API is used to handle operations that take time, such as fetching data from a server, without stopping the execution of the program. It works using async and await, making asynchronous code easier to read and manage than Promises. 

e.g : 
async function getData() {
    let response = await fetch("https://api.example.com/data");
    let data = await response.json();
    console.log(data);
}

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

9. TypeScript : TypeScript is a superset of JavaScript that adds static typing, interfaces, classes, and better error checking during development. It helps write safer and more maintainable code.

e.g. :
let username: string = "Prabhav";
let age: number = 22;

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


10. Merge Interface : Interface merging happens when multiple interfaces with the same name are declared. TypeScript automatically combines them into one single interface. 

e.g. :
interface User {
    name: string;
}

interface User {
    age: number;
}

const person: User = {
    name: "Prabhav",
    age: 22
};


-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


11. Declaration Merging : Declaration Merging is a feature in TypeScript where multiple declarations with the same name are merged into one definition. It commonly works with interfaces, namespaces, and enums.

e.g. :
interface Student {
    name: string;
}

interface Student {
    course: string;
}

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

12. Enums : Enums (Enumerations) are used to define a set of named constant values. They improve code readability and make values easier to manage.

e.g. :
enum Role {
    Admin,
    User,
    Guest
}

let myRole: Role = Role.Admin;
console.log(myRole); // 0


13. Tuples : Tuples are a special type of array in TypeScript where the number of elements and their data types are fixed in a specific order. 

e.g. :
let user: [string, number] = ["Prabhav", 22];


-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


14. Difference Between Array and Tuple : 

Array	                                                            Tuple
Stores multiple values of same type	                                Stores fixed values of different types
Length can change	                                                Length is fixed
Example: number[]	                                                Example: [string, number]

e.g. :
let arr: number[] = [10, 20, 30];
let tuple: [string, number] = ["Age", 22];


-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


15. Generics : Generics allow us to create reusable components that work with different data types without losing type safety. 

e.g. :
function identity<T>(value: T): T {
    return value;
}

console.log(identity<string>("Hello"));
console.log(identity<number>(100));

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

16. Constraint Generics : Constraint Generics are used when we want to limit the type that a generic can accept using the extends keyword. This ensures only specific types are allowed.

e.g. :
function printLength<T extends { length: number }>(item: T): void {
    console.log(item.length);
}

printLength("Hello");
printLength([1, 2, 3]);
